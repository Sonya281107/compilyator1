
# Спецификация языка программирования

## 1. Общая характеристика языка

Разрабатываемый язык представляет собой компилируемый язык программирования, синтаксически и концептуально близкий к упрощённой 
смеси C и Rust. От языка C он наследует императивную модель вычислений, функции, условные и циклические конструкции, а также 
привычный стиль записи выражений. От Rust язык заимствует более строгий подход к типизации, явность преобразований, а также 
разделение переменных на изменяемые и неизменяемые. 

Цель языка — предоставить достаточно низкоуровневый и предсказуемый инструмент для написания программ, при этом избежать части 
проблем классического C-подхода, связанных с неявными преобразованиями, слабым контролем типов и неочевидным изменением состояния. 
Язык ориентирован на простую и прозрачную модель выполнения: программа состоит из набора объявлений функций, глобальных сущностей 
и последовательности операторов, которые компилятор может строго проверить ещё до запуска.

Язык использует статическую строгую типизацию. Это означает, что тип каждого выражения должен быть известен на этапе компиляции, 
а несовместимые операции приводят к ошибке компиляции. Неявные преобразования между принципиально разными типами не допускаются; 
если программист хочет преобразовать значение, он обязан сделать это явно.

Дополнительно язык поддерживает ограниченный вывод типов. Если объявляется локальная переменная с инициализатором, и тип выражения 
определяется однозначно, компилятор может вывести тип автоматически. Однако вывод типов не должен ухудшать читаемость программы 
или создавать неоднозначность. Поэтому он применяется только в простых и строго определённых случаях. 


## 2. Система типов

Типовая система языка построена на принципе явности и минимальной неоднозначности. Все типы делятся на несколько основных категорий: 
целочисленные, вещественные, логический, символьный, а также специальный тип отсутствия значения.

### 2.1. Встроенные типы

В базовую спецификацию языка входят следующие встроенные типы:

* `i32` — 32-битное знаковое целое число;
* `i64` — 64-битное знаковое целое число;
* `u32` — 32-битное беззнаковое целое число;
* `u64` — 64-битное беззнаковое целое число;
* `f32` — 32-битное вещественное число;
* `f64` — 64-битное вещественное число;
* `bool` — логический тип со значениями `true` и `false`;
* `byte` — одиночный символ; 
* `void` — специальный тип, обозначающий отсутствие возвращаемого значения у функции

### 2.2. Строгость типизации

Язык является строго типизированным. Это означает следующие правила:

1. Операции допустимы только между совместимыми типами.
2. Присваивание возможно только тогда, когда тип выражения совпадает с типом переменной.
3. Арифметические и логические операции не смешиваются.
4. Любое преобразование между, например, `i32` и `f32` выполняется только явно.

Пример корректного кода:

let x: i32 = 10;
let y: i32 = 20;
let z: i32 = x + y;

Пример некорректного кода:

let x: i32 = 10;
let y: f32 = 2.5;
let z = x + y;    // ошибка: разные типы без явного преобразования


### 2.3. Явные преобразования типов

Преобразования типов в языке являются явными. Программист должен явно указать, что значение одного типа необходимо привести к 
другому. Это правило уменьшает число скрытых ошибок и делает семантику выражений более очевидной.

Пример:

txt
let x: i32 = 10;
let y: f32 = to_f32(x);

Форма записи преобразования будет реализована как встроенная функция (`to_f32(x)`) или как специальный синтаксис приведения 
типа. Главное требование спецификации — отсутствие неявного автоматического приведения между числовыми типами.


## 3. Переменные и изменяемость

Одной из ключевых особенностей языка является разделение переменных на неизменяемыеи изменяемые.

### 3.1. Неизменяемые переменные

По умолчанию переменная является неизменяемой. После инициализации её значение нельзя изменить. Такой подход делает код безопаснее
и упрощает анализ программы компилятором.

Пример:

let x: i32 = 5;
x = 10;   // ошибка: x неизменяема


### 3.2. Изменяемые переменные

Если переменная должна менять значение, это необходимо явно указать через ключевое слово `mut`.

Пример:

let mut counter: i32 = 0;
counter = counter + 1;


Таким образом, изменение состояния в программе всегда видно прямо в тексте, а не скрыто по умолчанию.

### 3.3. Вывод типа переменной

Для локальных переменных допускается ограниченный вывод типа. Если справа от оператора присваивания стоит выражение, тип которого 
известен однозначно, тип можно не указывать.

Пример:

let x = 10;        // x выводится как i32
let flag = true;   // flag выводится как bool

Если тип определить нельзя или существует неоднозначность, программист обязан указать его явно.

Пример:

let value;         // ошибка: невозможно вывести тип без инициализатора

Следовательно, вывод типов является вспомогательным механизмом, а не заменой явной типизации.


## 4. Выражения и операции

Выражение — это конструкция, вычисляющая значение некоторого типа. Язык поддерживает арифметические, сравнительные, логические 
и присваивающие операции.

### 4.1. Арифметические операции

Для числовых типов определены стандартные арифметические операции:

* `+` — сложение;
* `-` — вычитание;
* `*` — умножение;
* `/` — деление;
* `%` — остаток от деления.

Эти операции применимы только к совместимым числовым типам.

### 4.2. Операции сравнения

Операции сравнения возвращают значение типа `bool`:

* `==`
* `!=`
* `<`
* `<=`
* `>`
* `>=`

Операнды должны быть совместимых типов.

### 4.3. Логические операции

Для логических выражений поддерживаются:

* `&&` — логическое И;
* `||` — логическое ИЛИ;
* `!` — логическое НЕ.

Эти операции применяются только к значениям типа `bool`.

### 4.4. Приоритет операций

Приоритет операций близок к C-подобным языкам:

1. скобки;
2. унарные операции;
3. умножение, деление, остаток;
4. сложение и вычитание;
5. сравнение;
6. логическое И;
7. логическое ИЛИ;
8. присваивание.


### 5.1. Блок

Блок представляет собой последовательность операторов, заключённую в фигурные скобки:

{
    let x: i32 = 5;
    let y: i32 = x + 1;
}


Блок создаёт собственную область видимости.

### 5.2. Условный оператор

Условное выполнение задаётся конструкцией `if` / `else`:

if x > 0 {
    print_positive();
} else {
    print_non_positive();
}

Условие обязано иметь тип `bool`.

### 5.3. Цикл while

Цикл `while` выполняет блок, пока условие истинно:

while counter < 10 {
    counter = counter + 1;
}

### 5.4. Возврат из функции

Оператор `return` завершает выполнение функции и возвращает значение, если функция не имеет тип `void`.

return x + y;

## 6. Функции

Функция является основной единицей организации кода. Она имеет имя, список параметров, тип возвращаемого значения и тело.

### 6.1. Объявление функции

Общий вид функции:

fn add(a: i32, b: i32) -> i32 {
    return a + b;
}

Здесь:

* `fn` — ключевое слово объявления функции;
* `add` — имя функции;
* `a: i32, b: i32` — параметры;
* `-> i32` — возвращаемый тип;
* тело функции находится в блоке.

### 6.2. Тип возвращаемого значения

Если функция ничего не возвращает, используется тип `void`:

fn print_hello() -> void {
    // ...
}

Каждая функция должна либо явно возвращать значение нужного типа, либо, если она имеет тип `void`, завершаться без значения.

### 6.3. Параметры функций

Параметры функции типизируются явно. Неявный вывод типа параметров не допускается, поскольку это ухудшает читаемость интерфейса функции.

## 7. Область видимости и семантические правила

Каждая переменная видна только в пределах своей области видимости. Областью видимости являются блоки, тела функций и другие 
составные конструкции языка.

Основные семантические правила:

1. Переменная должна быть объявлена до использования.
2. В одной области видимости нельзя объявить две переменные с одинаковым именем.
3. Тип присваиваемого значения должен совпадать с типом переменной.
4. Условия в `if` и `while` обязаны иметь тип `bool`.
5. Функция должна возвращать значение заявленного типа.
6. Неизменяемая переменная не может быть переприсвоена.

## 8. Вывод

Таким образом, данный язык представляет собой статически типизированный компилируемый язык с императивной моделью выполнения, 
C-подобным синтаксисом и рядом более строгих правил, заимствованных из Rust-подхода: явной изменяемостью, запретом неявных 
преобразований и ограниченным выводом типов. 

Такая модель делает язык удобным для обучения, проектирования компилятора и дальнейшего расширения. Спецификация остаётся достаточно
простой, чтобы реализовать лексер, парсер и семантический анализатор, но при этом уже задаёт чёткие правила для типов, выражений, 
переменных и функций. В дальнейшем язык можно расширять пользовательскими структурами, массивами, модулями, системой владения 
памятью или ссылками, не нарушая базовых принципов, заложенных в данной версии.
